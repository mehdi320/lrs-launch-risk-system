## SECTION 6 — LE SCORING (comment ça marche)
**Type Carrd : Container avec liste**

### Titre
```
Un score /20 sur 4 axes critiques
```

### Sous-titre
```
LRS note uniquement ce qui est réellement présent dans ton contenu.
Pas de complaisance. Pas de notes gonflées.
```

### 4 blocs (liste ou colonnes)

```
⚡ Hook /5
La force de ton premier contact :
headline, image hero, ouverture de ta pub.
Un mauvais hook = tout le reste est inutile.
```

```
💰 Offer /5
La clarté et la valeur perçue de ton offre :
prix, stack, garantie, urgence.
```

```
🏆 Trust /5
Ta crédibilité :
avis, témoignages, preuves sociales, badges.
```

```
🔗 Friction / Message Match /5
La fluidité du parcours :
cohérence pub ↔ landing, CTA, nombre de frictions.
```

### Tableau décision (texte simple)
```
0 – 9  →  🚫 Do NOT launch    (risque élevé)
10 – 14 →  ⚠️ Test small budget  (50-200€/j max)
15 – 20 →  ✅ Ready to scale    (scaling progressif possible)
```

## SECTION 9 — FAQ
**Type Carrd : Accordion ou liste de textes**

```
Q : C'est quoi exactement LRS™ ?
R : LRS (Launch Risk System) est un outil d'audit pré-lancement pour le paid traffic.
Il analyse ta landing page et/ou tes publicités, te donne un score /20 et un verdict
de lancement avec un plan d'action concret.

Q : Ça marche pour quel type de business ?
R : E-commerce (produits physiques) et produits digitaux (formations, ebooks, SaaS, coaching).

Q : Quelles plateformes sont couvertes ?
R : Meta (Facebook/Instagram), TikTok, Google Ads, ou une analyse mixte.

Q : LRS remplace un media buyer ?
R : Non. LRS est un outil de pré-audit, pas de gestion de campagnes.
Il t'aide à vérifier que tes fondamentaux sont solides avant de lancer.

Q : Est-ce que mes données sont sauvegardées ?
R : L'historique de tes audits est sauvegardé localement dans ta session.
Rien n'est stocké sur des serveurs externes.
```
